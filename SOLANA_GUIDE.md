# 🚀 Инструкция по выживанию Solana Developer (Devnet/Build/Deploy)

Этот документ содержит «золотые правила» и технические решения, выработанные в ходе борьбы за стабильный билд вашего проекта.

## 🏝 1. Оазис Стабильности (Папка `deps/`)
Из-за глобального перехода библиотек Rust на `edition 2024`, ваш тулчейн (Rust 1.75.0) может ломаться. Мы создали защищенный контур:
- **Что это?** Локальные копии критических библиотек (blake3, borsh, RustCrypto).
- **Важно:** НИКОГДА не удаляйте папку `deps/` и не меняйте секцию `[patch.crates-io]` в корневом `Cargo.toml`. Это ваш «бронежилет» от поломок из интернета.

## 🛠 2. Правильный Билд (No-Error Workflow)
**Всегда** собирайте проект без генерации IDL, если возникают ошибки макросов:
```bash
# Идеальная команда для сборки
anchor build --no-idl
```
*Почему?* Anchor 0.30/0.31 часто конфликтует с `proc-macro2` на этапе IDL. Сборка самого байткода (`.so`) важнее для деплоя.

## 📏 3. Правило «Размера» (Перед деплоем)
Solana не расширяет место под программу автоматически. Чтобы не получить ошибку `account data too small`:
1. **Замерьте файл**: `ls -lh target/deploy/depin_pricing.so`
2. **Проверьте сеть**: `solana program show <ID> --url devnet`
3. **Расширьте (если надо)**: 
   ```bash
   solana program extend <ID> 50000 --url devnet
   ```
   *Всегда добавляйте с запасом (например, 50к байт).*

## 🔋 4. Авто-миграция (realloc)
Мы внедрили `realloc` в инструкцию `update_multiplier`. 
- **Как это работает:** Если ваш аккаунт стейта (`PriceHistory`) старый и маленький, программа САМА расширит его при первой же транзакции от AI-агента.
- **Условие:** У кошелька AI-агента должно быть немного SOL на оплату аренды места.

## 🛑 ЧЕГО ДЕЛАТЬ НЕЛЬЗЯ:
1. **`cargo update` без повода**: Это может обновить заблокированные библиотеки и сломать билд.
2. **Удаление `Cargo.lock`**: Этот файл — единственный гарант того, что версии библиотек не улетят в "будущее" (edition 2024).
3. **Деплой без проверки баланса**: Убедитесь, что `solana balance` > 1 SOL для спокойного деплоя.
